git 이슈
다른 곳에서 쓸땐 한 꼭 패치 시키고 쓰기 

같은 리포지토리 다른 곳에서 쓸때 혼용되는 일이 없게 끔 (자료가 사라지는 끔찍한 일이 일어날 수 있다)
작업전에 패치를 내려받고 소스트리로 보기 다른 커밋이 있는 지 없는지
git fetch -p를 해줘야 매치되서 사고없이 파일을 올릴수 있다.
git log --all --graph 이제까지의 commit 로그를 볼 수 있다.
git pull 위에 딴데서 올려진 최종 올라가 있는 걸로 내껄 덮어쓴다 최신껄로

git reset --hard (commit주소값) 가지치기가 되어 분기가 나뉘었을 때 이전(commit 주소값)으로 되돌릴수 있는 명령어

git alias 깃 명령어 사용자 설정 할수 있다


1. 사용자로 부터 2개의 문자열을 읽어서 같은지 다른지 확면에 출력하는 프로그램 작성
	ex)
		첫번째 문자열 : hello
		두번째 문자열 : world
		2개의 문자열은 다릅니다.
	-Equals 메서드를 사용하지 않을 것

	-힌트 Length를 비교 한다던지?

2. 5개의 음료 (콜라,물.스프라이트,주스,커피)를 판매하는 자판기 머신을 구현해보기

사용자가 1부터 5 사이의 숫자를 입력함
선택한 음료를 출력함
1~ 5 이외의 숫자를 선택하면 오류 메시지 출력함
EX)
	콜라,물,스프라이트,주스,커피, 중에서 하나를 선택하세요. : 1
	콜라를 선택하였습니다.
	-프로그램 종료- 

힌트 switch 문이나 if/else문을 사용한다,

3. 배열 days[]를 아래와 같이 초기화 하고 배열 요소에 값을 다음과 같이 출력하는 프로그램 작성	
	- 배열 days[] 는 -> 31, 29, 31 ,30, 31,30,31,30,31,30,31
EX
	1월은 31일까지 입니다.
	2월은 29일까지 입니다.
	3월은 31일까지 입니다.

	힌트 배열의 초기화는 중괄호를 사용한다.
